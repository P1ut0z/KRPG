# KRPG Mature Knowledge Graph

The project knowledge graph is generated by:

```bash
python scripts/build_kg.py --amp-csv data/amp_dataset_clean.csv --out data/knowledge_base
```

`scripts/build_kg.py` now builds a mature graph in two layers:

1. Base design knowledge:
   AMP design properties, GO terms, pathogen classes, motifs, and design rules.
2. Data-derived knowledge:
   Curated AMP peptide nodes from `data/amp_dataset_clean.csv`, enriched with
   DRAMP assay/citation/structure fields and DBAASP database/PDB/modification
   metadata from `data/raw`.

## Files

- `entities.jsonl`: one JSON entity per line.
- `relations.jsonl`: one JSON relation per line.
- `kg_summary.json`: entity/relation counts and raw enrichment statistics.

## Main Entity Types

- `peptide`: AMP sequence with computed physicochemical properties.
- `database_record`: APD6, DRAMP, and DBAASP source records.
- `activity_class`: normalized activity labels such as antimicrobial,
  antibacterial, antifungal, antiviral, anticancer.
- `target_class`: broad target groups such as Gram-negative bacteria,
  Gram-positive bacteria, fungi, virus, cancer cells.
- `target_organism`: parsed organism or cell-line targets.
- `activity_assay`: DRAMP MIC/IC/MBC-style activity measurements.
- `toxicity_assay`: DRAMP hemolysis/cytotoxicity evidence.
- `literature_reference`: PubMed or reference-text evidence nodes.
- `structure_feature`: PDB, topology, stereochemistry, and structural metadata.
- `modification`: terminal modifications from DBAASP.

## Main Relation Types

- `mapped_to`: peptide to source database record.
- `derived_from`: peptide to source record or curated source.
- `has_activity_class`: peptide to normalized activity class.
- `active_against`: peptide or assay to target class/organism.
- `has_activity_assay`: peptide to measured activity assay.
- `has_toxicity_assay`: peptide to toxicity/hemolysis evidence.
- `supported_by`: peptide or database record to literature reference.
- `has_structure`: peptide to structure/PDB/topology feature.
- `has_modification`: peptide to terminal modification.

The runtime retrieval layer prefers this graph automatically when
`data/knowledge_base/entities.jsonl` and `relations.jsonl` exist.
