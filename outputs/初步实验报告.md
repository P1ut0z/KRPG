## 3. 各模块实现与验证结果

### 3.1 模块一：知识图谱检索模块

#### 实现内容

| 组件 | 文件 | 功能 |
|------|------|------|
| KnowledgeGraph | [knowledge_graph.py](../krpg/knowledge/knowledge_graph.py) | 实体-关系知识图谱，支持添加、查询、邻居遍历、序列化 |
| RAGRetriever | [rag_retriever.py](../krpg/knowledge/rag_retriever.py) | 基于设计目标的多维度知识检索，支持知识库搜索 |
| PromptBuilder | [prompt_builder.py](../krpg/knowledge/prompt_builder.py) | 结构化 Prompt 构建，支持多目标和反馈增强 |

#### 验证结果

- **知识图谱**: 构建了包含 13 个实体、9 条关系的默认 AMP 知识图谱
  - 实体类型: property(5), go_term(3), pathogen(2), motif(3)
  - 支持实体查询、关系查询、邻居遍历
- **RAG 检索**: 对"广谱、低毒、抗革兰氏阴性菌"目标检索到 12 条知识项
  - 知识库搜索返回 3 条相关结果（毒性、图谱统计、两亲性螺旋）
- **Prompt 构建**: 成功生成了 858 字符的结构化 Prompt

### 3.2 模块二：序列生成模块

#### 实现内容

| 组件 | 文件 | 功能 |
|------|------|------|
| AminoAcidTokenizer | [tokenizer.py](../krpg/generation/tokenizer.py) | 20 种标准氨基酸 + 5 个特殊 token 的词表 |
| KRPGGenerator | [model.py](../krpg/generation/model.py) | GPT式 Transformer (CausalSelfAttention + FFN) |
| SequenceGenerator | [generator.py](../krpg/generation/generator.py) | 训练引擎 + 多策略生成 (top-k, top-p, temperature) |

#### 模型参数

- **轻量版**: d_model=64, n_heads=2, n_layers=2, d_ff=128 → **72,768 参数 (0.07M)**
- **全量版**: d_model=256, n_heads=4, n_layers=6, d_ff=1024 (配置就绪)

#### 验证结果

- **Tokenizer**: 25 词表大小，encode/decode 往返测试通过
- **训练**: 5 个样本，5 个 epoch，loss 从 3.21 降至 2.53（持续下降，训练有效）
- **生成**: 成功生成 5 条候选序列（如 `FSKGIAG`, `LGKDPGGGRRWFIYQKTKKA` 等）
- **端到端生成**: 8 条候选序列，长度 1-20 aa

### 3.3 模块三：计算验证模块

#### 实现内容

| 组件 | 文件 | 功能 |
|------|------|------|
| RuleBasedAMPPredictor | [amp_predictor.py](../krpg/validation/amp_predictor.py) | 基于理化规则的 AMP 活性评分 |
| AMPPredictor | [amp_predictor.py](../krpg/validation/amp_predictor.py) | 神经网络 AMP 预测器（待训练） |

| FeedbackOptimizer | [feedback.py](../krpg/validation/feedback.py) | 结果分析、模式提取、约束更新建议 |

#### 验证结果

**AMP 活性预测**:
| 序列 | AMP Score | 候选? |
|------|-----------|-------|
| KWLKKIGAVLKVL | 0.750 | ✅ |
| GFKRIVQRIKDFL | 1.000 | ✅ |
| LYIAKLLKRFN | 1.000 | ✅ |
| WWWWWWWWWW | 0.250 | ❌ |
| GAGAGAGAGA | 0.400 | ❌ |

**毒性预测**: GFKRIVQRIKDFL 和 LYIAKLLKRFN 被判定为低毒
**稳定性**: 多数序列稳定性中等（0.40-0.50），WWWWWWWWWW 稳定性最高（0.70）
**新颖性**: LYIAKLLKRFN 等序列与已知 AMP 相似性低，具有新颖性

## 4. 当前结果的局限性

### 4.1 模型训练不足
- 仅使用 5 条 AMP 序列训练了 3-5 个 epoch，模型远未收敛
- 生成的序列质量较低（部分序列过短或氨基酸组成异常）
- 需要更多数据和更长时间的训练才能产生有生物学意义的序列

### 4.2 验证规则简单
- AMP 活性预测基于简单的理化规则（电荷、疏水性、长度），未使用训练好的 ML 模型
- 毒性预测基于基序匹配，精度有限
- 稳定性评估未考虑二级结构信息

### 4.3 知识图谱规模小
- 当前知识图谱仅包含 13 个实体，远未覆盖 AMP 领域的完整知识
- 未接入真实 AMP 数据库（如 DRAMP, APD3, CAMP 等）
- 文献知识尚未纳入检索系统

### 4.4 无结构预测和对接模拟
- 未集成 AlphaFold/ESMFold 等结构预测工具
- 未进行分子对接模拟
- 缺少三维结构层面的验证

### 4.5 闭环优化未充分验证
- 反馈优化模块已实现，但仅进行了单轮测试
- 多轮迭代优化的效果尚未验证

## 5. 后续服务器全量实验计划

### 5.1 数据准备阶段
| 任务 | 说明 | 优先级 |
|------|------|--------|
| 收集 AMP 数据库 | 从 DRAMP、APD3、CAMP、DBAASP 等数据库收集 AMP 序列 | P0 |
| 数据清洗与标准化 | 去重、统一标签、过滤无效序列 | P0 |
| 构建 AMP 属性数据集 | 整理活性、毒性、稳定性标签 | P1 |
| 扩充知识图谱 | 加入更多实体和关系，接入 GO 数据库 | P1 |

### 5.2 模型训练阶段
| 任务 | 说明 | 预计 GPU 时间 |
|------|------|---------------|
| 训练 GPT 生成模型 | d_model=256, n_layers=6, 100 epochs, 全量数据 | ~4-8 小时 |
| 训练 AMP 分类器 | 基于已知 AMP/非AMP 序列训练二分类模型 | ~1-2 小时 |
| 训练毒性预测模型 | 基于已知毒性数据训练回归/分类模型 | ~1-2 小时 |

### 5.3 全量生成与评估
| 任务 | 说明 |
|------|------|
| 多目标 Prompt 生成 | 针对 5-10 种设计目标各生成 500 条候选序列 |
| 多维度评估 | AMP 活性、毒性、稳定性、新颖性综合评分 |
| 结构预测 | 对 Top-50 候选进行 ESMFold 结构预测 |
| 相似性分析 | 与已知 AMP 数据库进行全库比对 |

### 5.4 闭环优化
| 轮次 | 说明 |
|------|------|
| Round 1 | 初始知识 Prompt → 生成 → 评估 |
| Round 2 | 根据 Round 1 反馈调整约束 → 重新生成 |
| Round 3 | 进一步优化 Prompt → 生成高质量候选集 |

### 5.5 服务器部署命令

```bash
# 训练生成模型
python scripts/run_server.py --mode train --epochs 100 --batch_size 64 --d_model 256 --n_layers 6

# 生成候选序列
python scripts/run_server.py --mode generate --n_sequences 500 --checkpoint outputs/checkpoints/krpg_model_epoch100.pt

# 评估候选序列
python scripts/run_server.py --mode evaluate --input_file outputs/generated/candidates_*.json

# 完整流水线
python scripts/run_server.py --mode full --epochs 100 --n_sequences 500
```
